# AI向けドキュメント

- 問題が発生したとき、「問題は〇〇のせいです」と決め打ちして解決しようとしないこと。問題にあたりをつけるのは良いが、それが違ったときにどんどん無駄なコードを増やして複雑な方向に行ってはならず、もとの仮説が正しかったのかどうか確認を先にする必要がある
- 余計な機能をつけない。Nice to haveをつけない。ある程度充実したユーザー指示のもとでは、まずその要件通りに作れるかどうか考える。もし要求通りに作ったときに明らかにfragileである、あるいはカバーしきれないケースが有り得そうなときには、いきなり実装に反映するのではなく、ユーザーに確認を取る。
- 上記と似ているが、ユーザーが具体的なHOWを指定しているにもかかわらず、あなたが見つけた技術制約に従って、ユーザー指示を無視した実装を行うことも禁止である。「ユーザーの指定したやり方では実現できないので、代わりの方法で進めちゃいました！」というのは気が利かない。本当に気が利く人は、勝手に進めるんじゃなくて、その問題点をちゃんと報告して合意形成してから次に進める人だ。
